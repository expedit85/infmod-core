Ajuda da interface REST do relógio

O formato padrão de data e hora utilizado é o definido para UTC pelo W3C para xsd:dateTime:

yyyy-MM-dd'T'HH:mm:ss'Z'

Exemplo:

2011-12-21T14:57:00Z

Comandos REST aceitos

  • / e /help

    • Exibe esta ajuda.

  • /admin

    • Página de administração via Web

  • /datetime

    • Mostra o horário atual simulado no formato padrão.

  • /milliseconds

    • Mostra o horário atual simulado em milissegundos

  • /info

    • Retorna um XML com várias informações do relógio.

  • /configure

    • Configura o relógio com base em três parâmetros: startdate, finaldate e speedrate:

    ParâmetroValor
    startdateData inicial no formato padrão.
    finaldateData final no formato padrão.
    speedrateTaxa de atualização do relógio. Ex. 60 representa que passam 60s a cada 1s.

    Duas combinações de parâmetros são aceitas:

    1. Todos os parâmetros: configuração global que reinicia o relógio.
    2. Apenas speedrate: altera apenas a taxa de atualização do relógio atualmente em execução.

  • /restart

    • Reinicia o relógio com base na sua configuração atual

  • /isalive

    • Retorna true enquanto relógio estiver em execução, ou seja, não alcançou a data final (finaldate)
